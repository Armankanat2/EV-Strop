# Документы проекта

Дата обновления структуры: `2026-06-20`

## Назначение

Папка `docs/` больше не используется как общий склад файлов.

Теперь документы разложены по тематическим разделам, чтобы:

- быстрее находить нужный материал;
- безопаснее вносить изменения в структуру и нумерацию;
- не смешивать курс, тесты, production-реестры, концепты и нормативную базу.

## С чего начинать

Если нужно понять, как устроен курс:

- открыть `docs/01_структура_курса/`
- потом `docs/04_production_и_реестры/`

Если нужно менять структуру или нумерацию:

- открыть `docs/00_навигация_и_изменения/README_система_управления_изменениями_2026-06-20.md`

Если нужно работать с тестами:

- открыть `docs/03_тесты_и_разбор_ошибок/`

Если нужно искать исходные НПА и опорные материалы:

- открыть `docs/06_нпа_и_источники/`

## Практический принцип

Новые документы лучше класть не в корень `docs/`, а сразу в нужный раздел.

Если непонятно, куда положить файл:

- `структура курса` -> `01_структура_курса`
- `наполнение модуля` -> `02_контент_модулей`
- `тесты и разбор ошибок` -> `03_тесты_и_разбор_ошибок`
- `production и реестры` -> `04_production_и_реестры`
- `концепты и согласование` -> `05_концепты_и_согласование`
- `НПА и источники` -> `06_нпа_и_источники`
- `правила изменений, журналы, шаблоны` -> `00_навигация_и_изменения`

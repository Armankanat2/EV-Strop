# Операционный план медиапроизводства `ЭВ Стропальщик`

Дата фиксации: `2026-06-27`

## Назначение

Это основной живой документ по медиаподготовке презентации.

Он нужен, чтобы в одном месте видеть:

- что уже реально готово по физическим файлам;
- что собрано в текущей логике презентации;
- что еще только рекомендуется к сборке;
- откуда берем материал;
- какие задачи нужно выполнить дальше.

## Подтвержденные правила работы

1. Рабочий комплект документов ведем прямо в `ПРОЕКТ ЭВ СТРОПАЛЬЩИК/docs/04_production_и_реестры`.
2. `ПРОЕКТ ЭВ СТРОПАЛЬЩИК` фиксируется как чистый `production-проект`.
3. Внешний проект накопления называется `База знаний и материалов для ДПО`.
4. Внутри `ПРОЕКТ ЭВ СТРОПАЛЬЩИК` держим только материалы, которые прямо нужны для выпуска курса.
5. Инструкции, нормативка, документация, сравнение с чужими программами и общая экспертная база живут во внешнем проекте.
6. Связь между проектами осуществляется по отдельному регламенту обмена.
7. По охвату используем два контура:
   - `истина` - то, что уже собрано или подтверждено реальным результатом;
   - `рекомендация` - то, что еще предстоит собрать по структуре курса.
8. Поле `Срок` пока оставляем пустым.
9. Даты готовности вернемся заполнять после `2026-07-02`, когда будут согласования с руководителем.
10. В список `готово` включаем только реально существующие файлы в `assets`.
11. Формулировка `готово к наполнению` не считается готовым материалом.

## Главное правило актуальности

- этот файл хранит текущее состояние;
- если структура презентации изменилась, сначала обновляем этот план;
- потом фиксируем изменение в журнале обновлений;
- финальное решение считается закрепленным после твоего подтверждения.

## Ограничение на правки линейной структуры

Для текущего прохода сборки принимаем отдельное правило:

- до `S012` включительно линейную структуру не меняем;
- участок `S001-S012` считаем зафиксированной базой;
- все изменения линейного маршрута вносим только после `S012`, начиная с `S013`.

## Как читать поле `Контур`

- `истина` - есть конкретный результат: физический файл, уже используемый или готовый к использованию в презентации;
- `рекомендация` - задача следует из структуры курса, но медиа еще нужно подготовить или подтвердить;
- `снято` - задача больше не нужна после правки структуры.

## Универсальное правило для любого проекта ДПО

При открытии нового проекта ДПО работаем так:

1. Сверяем текущую структуру презентации.
2. Отделяем `истину` от `рекомендации`.
3. В `готово` заносим только реальные файлы из `assets`.
4. Все, чего не хватает для сборки, заводим как рекомендации с пустым сроком или с согласованной датой.
5. После каждой правки структуры сначала обновляем план, потом журнал.

## Правило обновления после правки структуры

Если ты меняешь структуру презентации:

1. В этом плане меняем `Контур`, `Слайд / узел`, `Что нужно подготовить` и `Статус`.
2. Если задача больше не нужна, ставим `снято`.
3. Если один слайд разделился на несколько, создаем отдельную строку на каждый медиа-объект.
4. Если появился реальный файл в `assets`, переводим строку из `рекомендация` в `истина`.

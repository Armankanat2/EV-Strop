# Рабочая карта наполнения: Тема 2.1 `Строповка грузов`

Дата фиксации: `2026-06-23`

## Назначение

Это прикладной production-документ по подтеме `2.1. Строповка грузов`.

Он нужен, чтобы по блоку `S008-S024` сразу было видно:

- что идет в линейной последовательности;
- где нужны обычные выносы `P`;
- где уже есть готовые тестовые выносы разбора ошибок;
- какие визуалы уже готовы, а какие еще нужно собрать;
- что остается на экране;
- что уходит в озвучку;
- как устроена навигация.

## Линейный маршрут блока

Основная линейка блока:

`S008 -> S009 -> S010 -> S011 -> S012 -> S013 -> S018 -> S019 -> S019A -> S020 -> S021 -> S022 -> S023 -> S024`

Это обязательный путь для каждого слушателя внутри подтемы `2.1`.

## Карта выносов блока

### Выносы от `S011`

- `S011-P01` -> возврат в `S011`
  Пояснение по центру тяжести и устойчивости груза.

### Выносы от `S012`

- `S012-P01` -> возврат в `S012`
  Быстрая справка по видам стропов и грузозахватных приспособлений.

### Выносы от `S013`

- `S013-P01` -> возврат в `S013`
  Что считается браковкой, износом и запретом на применение.

### Выносы от `S018`

- `S018-P56-01` -> возврат в `S018`
  Углубление для `5-6 разряда`: сложные и ответственные грузы.

- `S023-P01...P12` -> возврат в `S023`
  Разбор ошибок по неверным ответам промежуточного теста.

## Что считаем обязательным в первой сборке

Для первого рабочего прохода обязательны:

- `S008-S024` как целостная линейка;
- понятная навигация `Далее / Назад`;
- экран теста `S023`;
- возвраты из `S023-P01...P12` обратно в `S023`.

Желательно сразу подготовить:

- `S011-P01`
- `S012-P01`
- `S013-P01`
- `S018-P56-01`

Отдельное правило:

- `S023-P01...P12` уже обеспечены готовыми основными `png` и должны рассматриваться как не теоретический резерв, а как реальный production-актив для первой рабочей сборки теста.

## На что опираемся при сборке линейки `S008-S022`

Для основной линейки уже зафиксированы опорные визуальные решения:

- `S011` <- логика `S023-P03` и частично `S023-P09`
- `S012` <- общая типология из content-пакета модуля `1`
- `S013` <- логика осмотра и браковки, связанная с `S023-P06`
- `S018` <- обязательная основная схема правильной зацепки

- `S021` <- логика `S023-P07` и `S023-P11`

Практический вывод:

- блок `S023` уже не просто тестовый хвост, а готовый банк визуальных решений для всей подтемы `2.1`.

## Что еще нужно дособрать

Для линейных экранов `S008-S022` еще нужно отдельно упаковать:

- фото или набор карточек по типам грузов для `S010`;
- сравнительную таблицу по видам стропов для `S012`;
- экран `исправно / неисправно` для `S013`;
- главный экран правильной зацепки для `S018`;
- итоговую общую композицию по типовым схемам для `S022`.

Особенно важно не откладывать:

- `S018`, потому что это центральный экран правил строповки;
- `S019`, потому что он связывает риски с уже готовыми тестовыми визуалами;
- `S019A`, потому что без него теряется важное практическое правило по длинномеру и удавке;
- `S020`, потому что от него логически зависит весь тест `S023`.

# Production-реестр курса `ЭВ Стропальщик`

Дата фиксации: `2026-06-18`

## Назначение

Это производственный реестр по основному маршруту курса `S001-S057`.

Он нужен, чтобы по каждому слайду сразу видеть:

- что готовим в `текст`;
- что уходит в `озвучку`;
- какой нужен `визуал`;
- где нужен `видео`-материал;
- где собирается `тест`.

## Как читать реестр

- `Текст` — что должно быть на экране.
- `Озвучка` — что раскрываем голосом, без перегруза экрана.
- `Визуал` — основной тип графики или фото.
- `Видео` — `нет`, `реком.`, `обязательно`.
- `Тест` — `нет`, `вопрос`, `мини-тест`, `итоговый тест`, `разбор ошибки`.

## Отдельные производственные акценты

### 1. Видео, которое лучше заложить сразу

- `S014` — правильная строповка по шагам;
- `S027` — ручные сигналы и взаимодействие;
- `S044` — частные случаи нарушений / разбор ЧП;
- `S015` и `S028` — как рекомендованные короткие вставки, если будет ресурс.

### 2. Тестовые узлы курса

- количество промежуточных тестов на этом этапе не фиксируем жестко; ориентир — `до 5` узлов по мере наполнения курса;
- в текущем каркасе уже закреплены `4` базовых промежуточных теста;
- `S019` — мини-тест по теме `2.1`;
- `S032` — мини-тест по теме `2.2`;
- `S045` — мини-тест по теме `2.3`;
- `S052` — мини-тест по теме `3`;
- допускается добавление `5-го` промежуточного теста после наполнения, если появится отдельный смысловой узел, который лучше проверять отдельно;
- `S054-S056` — итоговый тестовый блок курса.

### 3. Следующий логичный документ

На основе этого реестра можно отдельно собрать:

- реестр `медиафайлов` с именами будущих файлов;
- реестр `озвучки` с хронометражем по слайдам;
- реестр `тестовых вопросов` по всем контрольным узлам курса.

### 4. Механизм разбора ошибки

- при неправильном ответе пользователь уходит не просто на экран `неверно`, а на отдельный поясняющий слайд;
- такой слайд объясняет: `в чем ошибка`, `какой ответ правильный`, `на какое правило курса нужно опереться`;
- после пояснения обязательно есть обратная ссылка `Вернуться к тесту`;
- для промежуточных тестов используем формат вроде `S019-P01`, `S032-P02`, `S045-P03`, `S052-P01`;
- это правило действует только в промежуточных тестах;
- на итоговый тест переход к поясняющему слайду не распространяется.

### 5. Логика итогового теста

- банк итогового теста: `40` вопросов;
- в одной попытке пользователь получает `20` вопросов;
- вопросы выбираются из банка случайно;
- если тест не сдан, при новой попытке порядок вопросов меняется;
- часть вопросов может повториться, но полная замена всех `20` вопросов не обязательна;
- оптимальная логика пересдачи: новая случайная выборка с частичным пересечением и новой последовательностью.

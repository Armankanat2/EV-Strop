# План генерации банка визуала: `ЭВ Стропальщик`

Дата фиксации: `2026-06-21`

## Назначение

Документ определяет, какие визуалы нужно подготовить для курса в первую очередь, чтобы:

- собрать единый модульный банк визуала;
- использовать одни и те же объекты в разных слайдах без дублирования;
- сначала генерировать master-assets, а уже потом собирать из них сцены;
- вести банк визуала последовательно и без повторов.

## Принцип сборки

Рабочая логика такая:

1. Сначала готовим отдельные сущности: `кран`, `стропальщик`, `ограждение`, `груз`, `площадка`.
2. Потом добавляем вариативность: позы, ракурсы, состояния, типы грузов и оснастки.
3. После этого собираем сценовые композиции для конкретных экранов курса.
4. Номер `VIS-xxxx` присваивается только готовому активу после сохранения в банк.

## Что делать в первую очередь

Рекомендуемая первая производственная волна:

1. `Кран`
2. `Стропальщик`
3. `Ограждение`
4. `Рабочая площадка`
5. `Труба`
6. `Пакет труб`
7. `Канатный строп`
8. `Текстильный строп`
9. `Крюк`
10. `Скоба`

Этого уже достаточно, чтобы начать собирать:

- титульные экраны;
- вводные рабочие сцены;
- базовую строповку;
- опасную зону;
- первые схемы по технологии работ.

## Набор поз и вариантов для людей

Для персонажа `стропальщик` лучше сразу делать не один актив, а мини-серию:

- нейтральная стойка;
- смотрит на груз;
- выполняет зацепку;
- подаёт сигнал;
- держит оттяжку;
- отходит из опасной зоны;
- показывает запрет / stop;
- участвует в инструктаже.

Для `машиниста` достаточно:

- в кабине;
- рядом с краном;
- в сцене взаимодействия со стропальщиком.

## Набор обязательных фонов

Чтобы не собирать каждый раз сцену с нуля, нужны отдельные фоновые основы:

- чистая строительная площадка;
- промышленная площадка с трубами;
- зона складирования;
- площадка для погрузки/разгрузки;
- площадка рядом с ЛЭП;
- площадка с уже размеченной опасной зоной.

## Форматы хранения

Рекомендуется вести банк так:

- отдельные объекты: `PNG` на прозрачном или чистом фоне;
- фоны и площадки: `PNG`;
- сценовые композиции: `PNG`;
- при необходимости анимации: исходник сцены + отдельные слои объектов;
- для реестра сразу фиксировать: тип, формат, расширение, фон, статус и сценарии использования.

## Правило production-потока

Для каждого нового актива:

1. Сначала фиксируем его в этом плане как `GEN-xxx` или `SCN-xxx`.
2. После фактической подготовки заносим в банк визуала с номером `VIS-xxxx`.
3. После назначения на экран курса добавляем строку использования `USE-xxxx`.

## Вывод

Базовый каркас банка визуала для курса должен строиться вокруг пяти главных сущностей:

- `кран`
- `стропальщик`
- `ограждение`
- `рабочая площадка`
- `груз / стропы / оснастка`

Если этот набор собрать аккуратно и единообразно, дальше большинство экранов курса можно будет делать через сборку сцен, а не через создание каждого визуала с нуля.

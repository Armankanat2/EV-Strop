## 4. Правило работы тестов

### Промежуточные тесты

- применяются в `S019`, `S032`, `S045`, `S052`;
- при ошибке пользователь уходит на отдельный поясняющий слайд `P`;
- после пояснения обязательно возвращается в исходный тест;
- формат возврата: `S019-Pxx -> S019`, `S032-Pxx -> S032`, `S045-Pxx -> S045`, `S052-Pxx -> S052`.

### Итоговый тестовый блок

- `S054` и `S055` работают как входной и переходный контрольный контур;
- `S056` работает как итоговый экзаменационный узел;
- в `S056` не используем поясняющие подвальные слайды;
- рабочее правило `S056`: банк `40` вопросов, в попытке `20`, при пересдаче новая выборка и новый порядок.

## 7. Что остается незакрытым

### По тестам

- заполнить `S054-Q01...Q06`;
- заполнить `S055-Q01...Q04`;
- собрать содержательный банк `S056-Q01...Q40`.

### По линейной сборке

- перенести зафиксированные экранные тезисы и озвучку в рабочие `PPTX`;
- собрать обязательные видео-вставки для `S014`, `S027`, `S044`;
- решить, нужны ли рекомендованные видео для `S015` и `S028`.

### По визуалам второго уровня

- подключать editable-мастера `pptx` только для сложных схем;
- подподвалы `PP01` открывать только там, где основной экран действительно перегружен.

## 8. Рекомендуемый следующий production-цикл

1. Собрать линейную презентацию `S001-S057` без новых развилок.
2. Встроить в нее все готовые визуалы `S019-Pxx`, `S032-Pxx`, `S045-Pxx`, `S052-Pxx`.
3. Закрыть содержанием тестовый блок `S054-S056`.
4. После первой сборки проверить, где реально нужны `PP01` и editable-мастера.

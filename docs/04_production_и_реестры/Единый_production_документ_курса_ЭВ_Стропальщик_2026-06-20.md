## 4. Правило работы тестов

### Промежуточные тесты

- применяются в `S023`, `S036`, `S049`, `S056`;
- при ошибке пользователь уходит на отдельный поясняющий слайд `P`;
- после пояснения обязательно возвращается в исходный тест;
- формат возврата: `S023-Pxx -> S023`, `S036-Pxx -> S036`, `S049-Pxx -> S049`, `S056-Pxx -> S056`.

### Итоговый тестовый блок

- `S058` и `S059` работают как входной и переходный контрольный контур;
- `S060` работает как итоговый экзаменационный узел;
- в `S060` не используем поясняющие подвальные слайды;
- рабочее правило `S060`: банк `40` вопросов, в попытке `20`, при пересдаче новая выборка и новый порядок.

## 7. Что остается незакрытым

### По тестам

- заполнить `S058-Q01...Q06`;
- заполнить `S059-Q01...Q04`;
- собрать содержательный банк `S060-Q01...Q40`.

### По линейной сборке

- перенести зафиксированные экранные тезисы и озвучку в рабочие `PPTX`;
- собрать обязательные видео-вставки для `S014`, `S018`, `S031`, `S048`;
- решить, нужны ли рекомендованные видео для `S019`, отдельная озвучка для `S019A` и видео для `S032`.

### По визуалам второго уровня

- подключать editable-мастера `pptx` только для сложных схем;
- подподвалы `PP01` открывать только там, где основной экран действительно перегружен.

## 8. Рекомендуемый следующий production-цикл

1. Собрать линейную презентацию `S001-S061` с новыми `S014-S017`.
2. Встроить в нее все готовые визуалы `S023-Pxx`, `S036-Pxx`, `S049-Pxx`, `S056-Pxx`.
3. Закрыть содержанием тестовый блок `S058-S060`.
4. После первой сборки проверить, где реально нужны `PP01` и editable-мастера.

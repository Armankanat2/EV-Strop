## 4. Правило работы тестов

### Промежуточные тесты

- применяются в `S018`, `S031`, `S044`, `S051`;
- при ошибке пользователь уходит на отдельный поясняющий слайд `P`;
- после пояснения обязательно возвращается в исходный тест;
- формат возврата: `S018-Pxx -> S018`, `S031-Pxx -> S031`, `S044-Pxx -> S044`, `S051-Pxx -> S051`.

### Итоговый тестовый блок

- `S053` и `S054` работают как входной и переходный контрольный контур;
- `S055` работает как итоговый экзаменационный узел;
- в `S055` не используем поясняющие подвальные слайды;
- рабочее правило `S055`: банк `40` вопросов, в попытке `20`, при пересдаче новая выборка и новый порядок.

## 7. Что остается незакрытым

### По тестам

- заполнить `S053-Q01...Q06`;
- заполнить `S054-Q01...Q04`;
- собрать содержательный банк `S055-Q01...Q40`.

### По линейной сборке

- перенести зафиксированные экранные тезисы и озвучку в рабочие `PPTX`;
- собрать обязательные видео-вставки для `S013`, `S026`, `S043`;
- решить, нужны ли рекомендованные видео для `S014` и `S027`.

### По визуалам второго уровня

- подключать editable-мастера `pptx` только для сложных схем;
- подподвалы `PP01` открывать только там, где основной экран действительно перегружен.

## 8. Рекомендуемый следующий production-цикл

1. Собрать линейную презентацию `S001-S056` без новых развилок.
2. Встроить в нее все готовые визуалы `S018-Pxx`, `S031-Pxx`, `S044-Pxx`, `S051-Pxx`.
3. Закрыть содержанием тестовый блок `S053-S055`.
4. После первой сборки проверить, где реально нужны `PP01` и editable-мастера.

# Схема тестовой логики курса `ЭВ Стропальщик`

Дата фиксации: `2026-06-18`

## Назначение

Это единый рабочий документ по логике тестов курса.

Он связывает в одной схеме:

- промежуточные тесты;
- механизм разбора ошибки;
- обратные переходы;
- итоговый тест;
- правило пересдачи итогового теста.

## Главный принцип

В курсе действуют два разных тестовых режима:

1. `Промежуточные тесты`
Проверяют отдельные смысловые блоки курса.
При неправильном ответе могут вести на поясняющий слайд с возвратом назад к тесту.

2. `Итоговый тест`
Проверяет курс целиком.
Перехода на поясняющие слайды при ошибке здесь нет.

## Карта тестовых узлов

На текущем этапе в каркасе курса закреплены такие тестовые узлы:

| ID | Тип | Блок курса | Разбор ошибки | Возврат |
|---|---|---|---|---|
| `S018` | промежуточный тест | Тема `2.1. Строповка грузов` | да | в `S018` |
| `S031` | промежуточный тест | Тема `2.2. Технология работы` | да | в `S031` |
| `S044` | промежуточный тест | Тема `2.3. Производство работ` | да | в `S044` |
| `S051` | промежуточный тест | Тема `3. Охрана труда` | да | в `S051` |
| `S053` | быстрая итоговая проверка | вход в итоговый блок | нет | не требуется |
| `S054` | ситуационный мини-тест | итоговый блок | нет | не требуется |
| `S055` | итоговый тест / квалификационный экзамен | итоговый блок | нет | пересдача внутри `S055` |

## Количество промежуточных тестов

Жестко количество промежуточных тестов пока не фиксируем.

Рабочее правило:

- ориентир — `до 5` промежуточных тестов;
- в текущем каркасе уже есть `4` базовых промежуточных теста;
- `5-й` промежуточный тест можно добавить позже, если в процессе наполнения появится отдельный смысловой узел, который лучше проверять отдельно.

## Логика промежуточного теста

Базовый сценарий промежуточного теста выглядит так:

`обучающий блок -> промежуточный тест -> результат -> следующий слайд темы`

Если ответ правильный:

- пользователь остается в тестовом экране;
- переходит дальше по сценарию темы.

Если ответ неправильный:

- пользователь уходит на отдельный поясняющий слайд;
- на поясняющем слайде видит, в чем ошибка;
- по кнопке `Вернуться к тесту` возвращается в исходный тестовый слайд.

## Формат разбора ошибки

Для промежуточных тестов используем `подвал` родительского теста.

Примеры:

- `S018-P01`
- `S031-P02`
- `S044-P03`
- `S051-P01`

Смысл такого слайда:

- коротко показать, какой ответ неверен;
- дать правильную логику;
- сослаться на правило или экран курса, на который нужно опереться.

Обязательное правило:

- слайд разбора ошибки всегда имеет кнопку `Вернуться к тесту`.

## Схема возвратов

Возвраты строятся строго по родительскому тесту:

- `S018-P01 -> S018`
- `S031-P02 -> S031`
- `S044-P03 -> S044`
- `S051-P01 -> S051`

Это нужно, чтобы пользователь не выпадал из тестового сценария и не терял контекст темы.

## Что не делаем в итоговом тесте

Для `S055` правило разбора ошибки не используем.

Это значит:

- неправильный ответ в итоговом тесте не ведет на отдельный поясняющий слайд;
- форматы вроде `S055-P01...P40` для разбора ошибок не применяются;
- итоговый тест живет как самостоятельный экзаменационный блок.

## Логика итогового теста `S055`

Для итогового теста принимаем такую схему:

- общий банк: `40` вопросов;
- в одной попытке: `20` вопросов;
- вопросы выбираются случайно из общего банка;
- порядок вопросов внутри попытки тоже случайный.

Если тест не сдан:

- запускается новая попытка;
- вопросы перемешиваются;
- порядок вопросов меняется;
- часть вопросов может повториться;
- полная замена всех `20` вопросов не обязательна.

Рабочая формула пересдачи:

`новая случайная выборка + новая последовательность + допустимое частичное пересечение`

## Роль `S053` и `S054`

Чтобы не путаться, разделяем их так:

- `S053` — быстрая проверка после курса, короткий вход в итоговый контрольный блок;
- `S054` — ситуационный мини-тест, который может оставаться тренировочным или переходным контрольным экраном;
- `S055` — полноценный итоговый тест / квалификационный экзамен.

Если в дальнейшей сборке решим упростить итоговый блок, возможны два варианта:

1. Оставить `S053` и `S054` как тренировочные экраны перед `S055`.
2. Свернуть `S053-S054` в более короткий вход и оставить главным контрольным узлом только `S055`.

## Короткая схема курса по тестам

`S018 -> S019`
при ошибке: `S018-Pxx -> S018`

`S031 -> S032`
при ошибке: `S031-Pxx -> S031`

`S044 -> S045`
при ошибке: `S044-Pxx -> S044`

`S051 -> S052`
при ошибке: `S051-Pxx -> S051`

`S053 -> S054 -> S055 -> S056`

Для `S055`:

- без поясняющих слайдов по ошибке;
- с новой случайной выборкой вопросов при пересдаче.

## Практический вывод

Для текущей сборки курса принимаем такие правила:

- промежуточные тесты можно развивать и добавлять по мере наполнения, ориентир — `до 5`;
- разбор ошибки действует только в промежуточных тестах;
- итоговый тест строится как случайная выборка `20` вопросов из банка `40`;
- пересдача итогового теста должна давать новый порядок вопросов и частично обновленную выборку.

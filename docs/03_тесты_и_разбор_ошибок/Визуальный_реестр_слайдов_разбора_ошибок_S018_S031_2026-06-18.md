# Визуальный реестр слайдов разбора ошибок `S018-P01...P12` и `S031-P01...P12`

Дата фиксации: `2026-06-18`

## Назначение

Это рабочий визуальный реестр для подвальных слайдов разбора ошибок в промежуточных тестах:

- `S018` — Тема `2.1. Строповка грузов`;
- `S031` — Тема `2.2. Технология работы стропальщика`.

Документ нужен, чтобы заранее закрепить:

- какой тип визуала нужен на каждом слайде;
- что именно должно быть показано;
- где достаточно простой графики;
- где нужна схема, знак или отдельный вложенный материал.

## Базовое правило для визуалов разбора ошибки

- визуал на таких слайдах должен быстро объяснять ошибку, а не просто украшать экран;
- для одного слайда лучше один главный визуальный объект, чем много мелких элементов;
- если ошибка пространственная или технологическая, приоритет у схемы;
- если ошибка процедурная, приоритет у чек-листа, карточки или короткой сравнительной графики;
- если ошибка связана со знаком, маркировкой или символом, приоритет у крупного набора знаков;
- визуал должен работать вместе с кнопкой `Вернуться к тесту`, не перегружая экран.

## Что можно собирать как единый пакет

Для ускорения работы визуалы удобно собирать не по одному, а пакетами:

1. Пакет `техсхемы строповки`
   Сюда входят `S018-P03`, `S018-P04`, `S018-P05`, `S018-P12`.

2. Пакет `знаки и маркировка`
   Сюда входят `S018-P02`, `S018-P09`, `S018-P11`.

3. Пакет `опасная зона и сопровождение`
   Сюда входят `S031-P07`, `S031-P08`, `S031-P11`.

4. Пакет `складирование и расстроповка`
   Сюда входят `S031-P09`, `S031-P10`, `S031-P12`.

## Практический вывод

Теперь у подвальных слайдов есть не только текстовая, но и визуальная рамка:

- понятно, где нужен чек-лист;
- понятно, где нужна схема;
- понятно, где нужен набор знаков;
- понятно, какие 8 слайдов критичны для наглядности в первую очередь.

Следующий удобный шаг — сделать production-реестр файлов для этих визуалов в формате `ID слайда / имя файла / тип файла / статус / источник / примечание`.

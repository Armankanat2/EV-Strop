# Схема нумерации слайдов: основной / подвал / подподвал

Дата фиксации: `2026-06-18`

## Назначение

Этот документ задает единый стандарт нумерации для:

- основного линейного курса;
- подвальных слайдов;
- подподвальных слайдов;
- отдельного контура `5-6 разряд`.

Эту схему нужно использовать в:

- названиях слайдов;
- таблицах сценария;
- именах файлов;
- ссылках в `PowerPoint`;
- навигации в `iSpring`.

## Главный принцип

Номер всегда должен сразу отвечать на три вопроса:

1. Это основной слайд или вынесенный.
2. От какого основного слайда он зависит.
3. Если это вложенный слайд, то на каком уровне вложенности он находится.

## 1. Основной линейный маршрут

Основные слайды нумеруем так:

`S001`
`S003`
`S004`

Правило:

- `S` = основной слайд;
- далее идет сквозной номер по всей линейной последовательности курса;
- используем трехзначный формат, чтобы нумерация не ломалась при росте курса.

Примеры:

- `S001` — титульный слайд
- `S003` — общая часть
- `S020` — правила строповки
- `S037` — промежуточный тест по разделу

## 2. Обычный `подвал`

Если с основного слайда есть переход в поясняющий слайд, используем такую форму:

`S020-P01`

Правило:

- `S020` = родительский основной слайд;
- `P` = подвальный слайд;
- `01` = первый подвальный слайд, связанный именно с этим основным слайдом.

Если у одного основного слайда несколько вынесенных пояснений:

`S020-P01`
`S020-P02`
`S020-P03`

Примеры:

- `S008-P01` — пояснение термина с основного слайда `S008`
- `S020-P02` — отдельная схема ко второму выносу со слайда `S020`

## 2A. `Подвал` для разбора ошибки в промежуточном тесте

Для промежуточных тестовых слайдов используем тот же формат `подвала`, но его смысл фиксируем отдельно:

`S023-P01`

Правило:

- `S023` = родительский слайд промежуточного теста;
- `P01` = слайд разбора ошибки по конкретному вопросу или по конкретной типовой ошибке;
- на таком слайде обязательно есть кнопка `Вернуться к тесту`.

Примеры:

- `S023-P01` — пояснение ошибки по одному из вопросов промежуточного теста темы `2.1`
- `S036-P03` — пояснение ошибки по вопросу 3 теста темы `2.2`

Важно:

- это правило распространяется только на промежуточные тесты;
- на итоговый тест механизм перехода на поясняющий слайд не распространяется.

## 3. `Подвал` для `5-6 разряда`

Для специального контура `5-6 разряд` используем отдельную маркировку:

`S020-P56-01`

Правило:

- `S020` = родительский основной слайд;
- `P56` = подвальный слайд специального контура `5-6 разряд`;
- `01` = первый слайд этого контура, привязанный к основному слайду `S020`.

Если для одного основного слайда есть несколько слайдов `5-6 разряд`:

`S020-P56-01`
`S020-P56-02`

Примеры:

- `S011-P56-01` — дополнительный материал для `5-6 разряда`
- `S026-P56-02` — второй специальный вынос для `5-6 разряда`

## 4. `Подподвал`

Если из подвального слайда нужен еще один уровень пояснения, используем такую форму:

`S020-P01-PP01`

Правило:

- `S020-P01` = родительский подвальный слайд;
- `PP01` = первый подподвальный слайд, открываемый именно из этого подвала.

Если у одного подвального слайда несколько вложенных слайдов:

`S020-P01-PP01`
`S020-P01-PP02`

## 5. `Подподвал` внутри контура `5-6 разряд`

Если второй уровень вложенности открывается из специального слайда `5-6 разряд`, используем такую форму:

`S020-P56-01-PP01`

Правило:

- `S020` = исходный основной слайд;
- `P56-01` = конкретный подвальный слайд `5-6 разряд`;
- `PP01` = первый вложенный слайд второго уровня.

Примеры:

- `S020-P56-01-PP01` — первая дополнительная схема внутри выноса `5-6 разряд`
- `S020-P56-01-PP02` — второй вложенный пример внутри того же выноса

## 6. Краткая таблица форматов

| Тип слайда | Формат номера | Пример |
|---|---|---|
| Основной линейный | `S001` | `S013` |
| Обычный подвал | `S001-P01` | `S013-P02` |
| Подвал `5-6 разряд` | `S001-P56-01` | `S013-P56-01` |
| Подподвал обычного подвала | `S001-P01-PP01` | `S013-P02-PP01` |
| Подподвал `5-6 разряд` | `S001-P56-01-PP01` | `S013-P56-01-PP02` |

## 7. Что писать на самих слайдах

Рекомендуемый служебный формат подписи:

- основной слайд: `S020`
- обычный подвал: `S020-P01`
- `5-6 разряд`: `S020-P56-01`
- подподвал: `S020-P56-01-PP01`

Если подпись выводится на экран, лучше ставить ее мелко:

- внизу справа;
- в служебной строке;
- или в заметках/слое, если не хотим перегружать интерфейс.

## 8. Правило возвратов

Нумерация должна сразу подсказывать обратный переход:

- `S020-P01` возвращается в `S020`
- `S020-P56-01` возвращается в `S020`
- `S020-P01-PP01` возвращается в `S020-P01`
- `S020-P56-01-PP01` возвращается в `S020-P56-01`

Для тестов это правило используем так же жестко:

- `S023-P01` возвращается в `S023`
- `S036-P03` возвращается в `S036`

Это нужно, чтобы после ошибки пользователь не терял попытку и не выпадал из сценария проверки.

## 9. Правило имен файлов

Имена файлов лучше строить от номера слайда.

Рекомендуемый шаблон:

`<номер>_<короткая-тема>_<тип>`

Примеры:

- `S020_pravila-stropovki_slide.pptx`
- `S020-P01_tsentr-tyazhesti_diagram.png`
- `S020-P56-01_otvetstvennye-gruzy_slide.pptx`
- `S020-P56-01-PP01_skhema-nagruzki_diagram.png`

## 10. Рабочее правило для сценариев и таблиц

В сценарных таблицах удобно держать отдельные колонки:

1. `ID слайда`
2. `Родительский слайд`
3. `Тип`
4. `Заголовок`
5. `Куда ведет ссылка`
6. `Куда ведет возврат`

Пример:

| ID слайда | Родитель | Тип | Заголовок | Переход | Возврат |
|---|---|---|---|---|---|
| `S020` | - | основной | Схемы строповки | `S021` / `S020-P01` | - |
| `S020-P01` | `S020` | подвал | Перечень схем строповки | `S020-PP01 ... S020-PP13` | `S020` |
| `S020-PP01` | `S020-P01` | подподвал | Схемы `1-2` | - | `S020-P01` |
| `S020-PP04` | `S020-P01` | подподвал | Схема `6` | - | `S020-P01` |

## 11. Правило безопасного перенумерования

В проекте допускается сдвигать, добавлять и удалять слайды, но делать это нужно не точечно, а как управляемое изменение всей связанной системы.

Базовое правило:

- сначала меняем основной линейный `ID` в master-документе;
- потом одним циклом обновляем все зависимые материалы;
- только после этого правим ссылки в `PowerPoint` и `iSpring`.

Что считаем master-уровнем:

- линейный маршрут `S001-Sxxx`;
- родительские тестовые узлы `S023`, `S036`, `S049`, `S056`, `S058-S060`;
- таблицы, где `ID` выступает как первичный ключ.

Что обязательно проверять после перенумерования:

- названия слайдов;
- переходы `вперед / назад`;
- возвраты из `подвала` и `подподвала`;
- тестовые связки `вопрос -> слайд разбора ошибки`;
- имена файлов `png/pptx`;
- production-реестры и master-таблицы.

Правило для обычной вставки нового линейного слайда:

1. определить, между какими `S-ID` он встает;
2. сдвинуть все следующие линейные `S-ID`;
3. вместе с родительским `S` сдвинуть и все связанные `P / PP / P56`;
4. обновить ссылки возврата и перехода;
5. отдельно перепроверить тестовые узлы, если сдвиг затронул `S023`, `S036`, `S049`, `S056`, `S058-S060`.

Правило для удаления линейного слайда:

- сначала удалить или переназначить все его дочерние `P / PP / P56`;
- затем закрыть разрыв в линейной нумерации;
- после этого проверить, не остались ли в документах и файлах старые `ID`.

Правило для подвальных слайдов:

- если меняется только состав `подвала`, а родительский `S-ID` не меняется, конфликт минимальный;
- если меняется родительский `S-ID`, все его дочерние `P / PP / P56` должны быть переименованы вместе с ним.

Практическое правило проекта:

- пока идет проектирование, перенумерование допустимо;
- после начала активной сборки курса лучше замораживать линейные `S-ID` пакетами;
- перед крупной сборкой в `PowerPoint/iSpring` нужно фиксировать рабочую версию нумерации и не менять ее без отдельного прохода по реестрам.

## 12. Чек-лист перенумерования

| Этап | Что проверить | Статус |
|---|---|---|
| До перенумерования | Понятно, какой `S-ID` добавляется, удаляется или сдвигается | `да / нет` |
| До перенумерования | Определен master-документ, с которого начинается правка | `да / нет` |
| До перенумерования | Понятно, какие тестовые узлы затрагиваются | `да / нет` |
| До перенумерования | Понятно, есть ли дочерние `P / PP / P56`, которые надо двигать вместе с родителем | `да / нет` |
| После перенумерования | Обновлены линейные переходы `далее / назад` | `да / нет` |
| После перенумерования | Обновлены возвраты из `подвала` и `подподвала` | `да / нет` |
| После перенумерования | Обновлены тестовые связки `вопрос -> слайд разбора ошибки -> возврат` | `да / нет` |
| После перенумерования | Обновлены имена файлов и production-реестры | `да / нет` |
| После перенумерования | Перепроверены ссылки в `PowerPoint` и `iSpring` | `да / нет` |
| После перенумерования | Не осталось старых `ID` в документах, таблицах и названиях файлов | `да / нет` |

Минимальное практическое правило:

- если хотя бы один пункт после перенумерования остается с ответом `нет`, нумерацию нельзя считать закрытой;
- если менялся тестовый узел, его лучше перепроверять отдельно как мини-сценарий целиком.

## 13. Практический вывод

Для проекта принимаем такую базовую схему:

- основной курс: `S001`
- обычный вынос: `S001-P01`
- специальный вынос `5-6 разряд`: `S001-P56-01`
- второй уровень вложенности: `S001-P01-PP01` или `S001-P56-01-PP01`

Этой схемы достаточно, чтобы:

- не путаться в навигации;
- удобно собирать кнопки возврата;
- быстро связывать слайды, медиа и сценарии между собой.

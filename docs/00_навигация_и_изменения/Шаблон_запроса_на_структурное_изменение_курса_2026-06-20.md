# Шаблон запроса на структурное изменение курса

Дата фиксации шаблона: `2026-06-20`

## Назначение

Этот шаблон используем перед любым заметным изменением структуры курса `ЭВ Стропальщик`.

Он нужен, чтобы до правки зафиксировать:

- что именно хотим изменить;
- зачем это делаем;
- какие блоки курса затронем;
- повлияет ли изменение на тесты, нумерацию и production-сборку.

## Когда использовать

Запрос оформляем, если планируется хотя бы одно из действий:

- добавить новый линейный слайд в маршрут `S001-S056`;
- удалить существующий слайд;
- перенести слайд в другой блок;
- разделить один слайд на несколько;
- объединить несколько слайдов в один;
- вставить новый тестовый узел или изменить место существующего;
- изменить состав темы, подтемы или итогового блока.

## Короткий шаблон

| Поле | Что заполнить |
|---|---|
| Дата | `YYYY-MM-DD` |
| Инициатор | Кто предлагает изменение |
| Название изменения | Короткое имя решения |
| Причина | Почему текущее решение уже не подходит |
| Что меняем | Какие `S-ID`, темы или блоки затрагиваются |
| Было | Текущее состояние |
| Предлагается | Новое состояние |
| Влияние на тесты | Какие тестовые узлы затрагиваются или `нет` |
| Влияние на нумерацию | Нужен ли сдвиг `S / P / PP / P56` |
| Влияние на production | Какие документы, файлы, `PPTX`, `iSpring` придется обновить |
| Риск | Низкий / средний / высокий |
| Решение | Принять / отложить / отклонить |
| Комментарий | Короткое пояснение |

## Мини-пример

| Поле | Пример |
|---|---|
| Дата | `2026-06-20` |
| Инициатор | методист |
| Название изменения | Разделить `S013` на два слайда |
| Причина | На одном экране слишком много логики и визуала |
| Что меняем | `S013-S014`, тема `2.1` |
| Было | Один слайд по правилам строповки |
| Предлагается | Отдельный слайд по базовым правилам и отдельный слайд по рискам |
| Влияние на тесты | косвенно влияет на `S018` |
| Влияние на нумерацию | да, потребуется сдвиг следующих `S-ID` |
| Влияние на production | master-таблица, production-реестр, ссылки в `PPTX` |
| Риск | средний |
| Решение | принять |
| Комментарий | Улучшает читаемость, но требует прохода по зависимым ID |

## Практический вывод

Этот шаблон нужен не вместо журнала, а до журнала.

Рабочая цепочка такая:

1. сначала оформляем запрос на структурное изменение;
2. потом принимаем решение;
3. затем вносим правку в структуру и нумерацию;
4. после этого фиксируем результат в соответствующих журналах.

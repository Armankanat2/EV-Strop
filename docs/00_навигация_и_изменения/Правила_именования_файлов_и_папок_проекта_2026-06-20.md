# Правила именования файлов и папок проекта

Дата фиксации: `2026-06-20`

## Назначение

Этот документ задает единые правила именования для проекта `ЭВ Стропальщик`.

Он нужен, чтобы:

- новые файлы не создавали хаос;
- документы, медиа и черновики назывались предсказуемо;
- по имени файла было понятно, что это и где это искать;
- структура проекта не расползалась снова.

## Главный принцип

Имя должно отвечать хотя бы на два вопроса:

1. Что это за материал.
2. К какому разделу, модулю или сценарию он относится.

Если имя не помогает быстро понять смысл файла, значит имя слабое.

## 1. Общие правила

- использовать понятные слова, а не случайные сокращения;
- не создавать файлы с именами вроде `новый документ`, `черновик`, `финал`, `финал2`, `последний финал`;
- для связанных серий использовать один и тот же шаблон;
- если файл входит в принятую систему `S / P / PP / P56`, это должно быть видно в имени;
- если файл временный или внешний, он должен лежать в правильной папке, а не в корне проекта.

## 2. Правила для папок

### Для верхнего уровня проекта

- верхнеуровневые папки называем коротко и по функции:
  `docs`, `notes`, `assets`, `presentations`, `knowledge_base`, `data`, `temp`

### Для служебных разделов

- если папки образуют управляемую последовательность, можно использовать числовой префикс:
  `00_`, `01_`, `02_`
- такой префикс используем только там, где реально важен порядок чтения или навигации.

Примеры:

- `docs/00_навигация_и_изменения`
- `docs/01_структура_курса`
- `notes/01_бриф_и_постановка`

### Для рабочих папок внутри раздела

- использовать нижнее подчеркивание `_`, если имя на русском и состоит из нескольких слов;
- не использовать лишние пробелы в служебных папках, если это не уже устоявшаяся согласовательная папка;
- английские технические слаги допустимы там, где они уже встроены в pipeline.

Примеры:

- `изменение_структуры`
- `03_библиотека_промтов`
- `module-01-stropovka-gruzov`

## 3. Правила для документов `.md`, `.docx`, `.pdf`

### Рабочие документы

Шаблон:

`<смысловое-имя>_<дата>`

Примеры:

- `Единая_карта_тем_и_модулей_стропальщик_2026-06-15.md`
- `Реестр_НПА_стропальщик_2026-06-11.md`
- `Правила_именования_файлов_и_папок_проекта_2026-06-20.md`

### Если документ входит в стабильную серию

Нужно сохранять общий формат серии.

Примеры:

- `Банк_вопросов_Модуль_1_Строповка_грузов_2026-06-16.md`
- `Таблица_шаблон_слайдов_Тема_3_Охрана_труда_2026-06-18.md`
- `Production_реестр_курса_ЭВ_Стропальщик_2026-06-18.md`

## 4. Правила для слайдов, визуалов и связанных файлов

### Основной слайд

Шаблон:

`S001_<короткая-тема>_<тип>`

Примеры:

- `S015_pravila-stropovki_slide.pptx`
- `S032_promezhutochnyi-test_slide.pptx`

### Подвал

Шаблон:

`S015-P01_<короткая-тема>_<тип>`

Примеры:

- `S015-P01_tsentr-tyazhesti_diagram.png`
- `S018-P03_tsentr-tyazhesti_diagram.png`

### Подподвал

Шаблон:

`S015-P01-PP01_<короткая-тема>_<тип>`

Пример:

- `S015-P01-PP01_skhema-nagruzki_diagram.png`

### Контур `5-6 разряд`

Шаблон:

`S015-P56-01_<короткая-тема>_<тип>`

Пример:

- `S015-P56-01_otvetstvennye-gruzy_slide.pptx`

## 5. Правила для черновиков и временных файлов

### Черновики презентаций

Черновые презентации кладем в:

- `presentations/drafts/`

Имя лучше строить так:

`<что-это>_<дата>` или `<что-это>_draft_<дата>`

Примеры:

- `karkas_prezentatsii_draft_2026-06-20.docx`
- `modul-1_scenario_draft_2026-06-20.pptx`

### Временные и внешние файлы

Если файл не относится к проекту напрямую, он должен лежать в:

- `temp/вне_проекта/`

Такие файлы не нужно подгонять под naming-стандарт курса.

Главное правило:

- они не должны лежать в корне проекта.

## 6. Что лучше не делать

- не использовать имена вроде `123.docx`, `test.md`, `новый 1.pptx`;
- не плодить `финал`, `финал_точно`, `финал_точно2`;
- не смешивать в одной серии русский стиль и случайные английские слова без причины;
- не использовать пробелы в служебных папках, если можно использовать `_`;
- не класть файлы в корень проекта, если для них уже есть подходящий раздел.

## 7. Быстрая памятка

### Для нового документа

- сначала выбрать правильную папку;
- потом назвать по схеме `смысл + дата`.

### Для нового слайда или визуала

- сначала определить `S / P / PP / P56`;
- потом добавить короткую тему;
- потом добавить тип файла.

### Для временного файла

- если он не постоянный, не класть его в корень;
- убрать его в `temp/` или `presentations/drafts/`.

## Практический вывод

Если сомневаешься, как назвать файл, используй самую простую формулу:

`что это + к чему относится + дата или ID`

Для этого проекта этого уже достаточно, чтобы не возвращаться к хаосу.

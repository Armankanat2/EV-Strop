# Шаблон решения по структурному изменению курса

Дата фиксации шаблона: `2026-06-20`

## Назначение

Этот шаблон нужен для короткой фиксации решения по структурному изменению курса `ЭВ Стропальщик`.

Его используем после оформления запроса на изменение, но до фактической правки структуры, нумерации и production-документов.

Шаблон запроса, к которому обычно привязывается это решение:
`docs/00_навигация_и_изменения/Шаблон_запроса_на_структурное_изменение_курса_2026-06-20.md`

## Когда использовать

Шаблон нужен, если по запросу требуется явно зафиксировать одно из решений:

- `принять`;
- `принять с условиями`;
- `отложить`;
- `отклонить`.

## Короткий шаблон

| Поле | Что заполнить |
|---|---|
| Дата | `YYYY-MM-DD` |
| Основание | Ссылка или название запроса |
| Название изменения | Короткое имя решения |
| Решение | `принять / принять с условиями / отложить / отклонить` |
| Условия | Что должно быть выполнено до внедрения или вместе с ним |
| Что меняем на практике | Какие блоки, `S-ID`, тесты или документы реально пойдут в правку |
| Что не меняем | Какие части курса остаются без изменений |
| Ответственный проход | Кто делает следующий рабочий проход |
| Куда фиксировать результат | Какие журналы и документы нужно обновить после внедрения |
| Комментарий | Короткое пояснение |

## Мини-пример

| Поле | Пример |
|---|---|
| Дата | `2026-06-20` |
| Основание | запрос на разделение `S013` |
| Название изменения | Разделить блок правил строповки на два экрана |
| Решение | принять с условиями |
| Условия | Сначала проверить влияние на `S018` и на все следующие `S-ID` |
| Что меняем на практике | `S013-S015`, master-таблицу, production-реестр, журнал структуры, журнал нумерации |
| Что не меняем | Темы `2.2`, `2.3`, блок `Тема 3` |
| Ответственный проход | методист + сборка |
| Куда фиксировать результат | журнал структуры и журнал нумерации |
| Комментарий | Изменение улучшает читаемость, но требует контролируемого сдвига маршрута |

## Практический вывод

Этот шаблон нужен как промежуточный слой между идеей и правкой.

Рабочая цепочка такая:

1. оформляем запрос на структурное изменение;
2. фиксируем решение по нему;
3. выполняем правку структуры и нумерации;
4. заносим результат в соответствующие журналы.

# Журнал изменений структуры курса `S001-S056`

Дата открытия журнала: `2026-06-20`

## Назначение

Это рабочий журнал для всех реальных изменений структуры курса `ЭВ Стропальщик` по основному линейному маршруту `S001-S056`.

Его используем, когда:

- добавляем новый линейный слайд в маршрут;
- удаляем слайд из утвержденной структуры;
- переносим слайд в другой блок;
- объединяем два слайда в один;
- делим один слайд на два и более;
- меняем состав темы, подтемы или тестового блока.

Основное правило:

- здесь фиксируем не только изменение `ID`, но и смысловое изменение_структуры;
- если изменилась логика маршрута курса, а запись сюда не внесена, изменение нельзя считать закрытым.

## Что считается структурным изменением

Структурным изменением считаем любое решение, которое влияет хотя бы на один из уровней:

- порядок прохождения `S001-S056`;
- состав темы `1`, `2`, `2.1`, `2.2`, `2.3`, `3`, `4`;
- место тестового узла внутри курса;
- состав экранов до или после теста;
- логику переходов между основными слайдами.

## Как закрывать запись

Запись можно считать закрытой только если:

- обновлена утвержденная структура или явно зафиксировано, что она не меняется;
- обновлена master-таблица курса;
- обновлен production-реестр курса;
- перепроверены тестовые узлы, если изменение затронуло `S018`, `S031`, `S044`, `S051`, `S053-S055`;
- при необходимости внесены изменения в журнал нумерации, если структура потянула за собой перенумерование.

## Практический вывод

Теперь у проекта есть отдельный живой журнал не только для нумерации, но и для структуры курса.

С этого момента:

- изменения `ID` лучше вести через журнал нумерации;
- изменения состава и порядка линейного маршрута лучше вести через этот журнал;
- если одно изменение затрагивает и структуру, и нумерацию, его нужно фиксировать в обоих журналах.

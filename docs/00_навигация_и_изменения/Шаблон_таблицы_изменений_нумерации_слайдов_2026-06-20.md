# Шаблон таблицы изменений нумерации слайдов

Дата фиксации шаблона: `2026-06-20`

## Назначение

Этот шаблон нужен для реальных проходов по перенумерованию в проекте `ЭВ Стропальщик`.

Его используем, когда:

- добавляем новый линейный слайд;
- удаляем существующий слайд;
- сдвигаем блок слайдов;
- меняем родительский `S-ID`, от которого зависят `P / PP / P56`;
- перепривязываем тестовые узлы и возвраты.

Основное правило:

- одна строка = одно управляемое изменение;
- если затронут целый диапазон, его лучше разбивать на несколько строк или на отдельный блок;
- изменения считаются закрытыми только после перепроверки ссылок, возвратов и файлов.

## Поля таблицы

- `Дата` — когда изменение внесено.
- `Инициатор` — кто инициировал изменение.
- `Тип изменения` — вставка, удаление, сдвиг, объединение, разделение, перепривязка.
- `Было` — исходный `ID` или исходная цепочка.
- `Стало` — новый `ID` или новая цепочка.
- `Что затронуто` — тема, модуль, диапазон слайдов, таблица или сценарий.
- `Дочерние P / PP / P56` — какие подвалы и подподвалы пришлось переименовать.
- `Тестовый узел` — если изменение затрагивает `S018`, `S031`, `S044`, `S051`, `S053-S055`, это фиксируем отдельно.
- `Файлы/реестры` — какие документы, имена файлов, `PowerPoint` или `iSpring` были обновлены.
- `Статус` — `план`, `в работе`, `закрыто`.
- `Комментарий` — короткое пояснение, зачем изменение вообще делалось.

## Рекомендуемый рабочий формат

### 1. Отдельная таблица на один проход

Если делаем один крупный проход по перенумерованию, удобно копировать этот шаблон в отдельный рабочий документ и вести изменения блоком.

### 2. Отдельный журнал по релевантным узлам

Если сдвиги происходят часто, можно вести накопительный журнал только по:

- линейным `S-ID`;
- тестовым узлам;
- подвальным слайдам с уже готовыми визуалами.

## Практический вывод

Этот шаблон нужен не для описания идеи, а для управляемой фиксации изменений.

Если перенумерование было, но не попало в такую таблицу, позже почти неизбежно появятся расхождения между:

- master-таблицей;
- production-реестрами;
- файлами визуалов;
- ссылками в презентации;
- тестовой логикой.

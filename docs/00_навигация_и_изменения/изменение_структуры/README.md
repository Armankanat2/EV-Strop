# Промты для изменения структуры

Дата фиксации: `2026-06-20`

## Зачем нужна эта папка

В этой папке лежат готовые промты для `4` самых частых ситуаций.

Идея простая:

- ты открываешь нужный файл;
- меняешь несколько строк в блоке `Что заполнить`;
- копируешь готовый промт целиком;
- вставляешь его в чат;
- дальше Codex сам проводит изменение по нашей системе документов.

## Какие промты здесь есть

1. `01_добавить_новый_линейный_слайд.md`
2. `02_убрать_линейный_слайд.md`
3. `03_разделить_один_слайд_на_два.md`
4. `04_добавить_подвальный_слайд.md`
5. `05_что_делаю_я_что_проверяешь_ты.md`
6. `06_боевая_памятка_4_команды.md`
7. `07_живые_примеры_4_команд.md`

## Как пользоваться

### 1. Если нужно добавить новый обычный слайд в основной маршрут

Открой:
`01_добавить_новый_линейный_слайд.md`

Используй, когда:

- в теме не хватает отдельного экрана;
- один смысловой шаг лучше вынести в новый `S`-слайд;
- новый слайд должен стоять между двумя существующими `S-ID`.

Что тебе сделать:

- вписать, между какими `S-ID` вставляем слайд;
- вписать название;
- вписать причину;
- при желании кратко описать содержание.

### 2. Если нужно убрать обычный слайд из основного маршрута

Открой:
`02_убрать_линейный_слайд.md`

Используй, когда:

- слайд лишний;
- два слайда дублируют друг друга;
- информацию лучше перенести в соседний экран.

Что тебе сделать:

- вписать `ID` удаляемого слайда;
- написать, почему он больше не нужен;
- указать, куда переносится его смысл, если переносится.

### 3. Если один слайд стал слишком тяжелым и его надо разделить на два

Открой:
`03_разделить_один_слайд_на_два.md`

Используй, когда:

- на одном экране слишком много текста;
- на одном экране смешаны два разных смысла;
- визуал и текст уже не помещаются нормально.

Что тебе сделать:

- вписать `ID` исходного слайда;
- коротко назвать первый новый кусок;
- коротко назвать второй новый кусок;
- написать причину разделения.

### 4. Если нужно добавить подвальный слайд

Открой:
`04_добавить_подвальный_слайд.md`

Используй, когда:

- основной `S`-слайд остается на месте;
- нужно отдельное пояснение, схема, термин, пример или разбор ошибки;
- нужно добавить `P`, `PP` или `P56`, не ломая основной маршрут.

Что тебе сделать:

- вписать родительский `S-ID`;
- указать, какой это тип: `P`, `PP` или `P56`;
- написать, что именно должно быть на выносе.

### 5. Если нужен совсем короткий алгоритм работы без длинных инструкций

Открой:
`05_что_делаю_я_что_проверяешь_ты.md`

Используй, когда:

- нужно быстро вспомнить порядок действий;
- не хочется перечитывать полный промт;
- нужно понять, что писать мне и что потом проверять руками.

### 6. Если нужны совсем короткие команды для копирования

Открой:
`06_боевая_памятка_4_команды.md`

Используй, когда:

- хочешь сделать минимум действий;
- не хочешь открывать длинные промты;
- тебе нужна одна короткая команда под каждую типовую ситуацию.

### 7. Если нужны уже заполненные примеры на реальных слайдах курса

Открой:
`07_живые_примеры_4_команд.md`

Используй, когда:

- хочешь не придумывать формулировку с нуля;
- хочешь просто взять похожий пример и слегка его поправить;
- тебе удобнее видеть команды сразу на живых `S-ID` курса.

## Самое короткое правило

- меняется основной маршрут `S001-S057` — бери один из первых трех промтов;
- добавляется только вынесенный слайд — бери четвертый промт.

## Что делает Codex после такого промта

Если промт заполнен нормально, Codex должен сам:

- провести изменение по системе проекта;
- обновить нужные документы;
- при необходимости сдвинуть нумерацию;
- обновить журналы;
- в конце кратко сказать, что именно поменялось.

# Живые примеры: 4 готовые команды по текущему курсу

Дата фиксации: `2026-06-20`

Ниже не шаблоны, а уже готовые примеры на реальных `S-ID` из текущей структуры курса.

## 1. Пример: добавить новый линейный слайд

```text
Добавь новый линейный слайд между S013 и S014. Название: Проверка груза перед выбором схемы. Причина: между правилами строповки и блоком рисков не хватает промежуточного шага проверки. Тест: нет. Проведи изменение по системе проекта и обнови все связанные документы.
```

## 2. Пример: убрать линейный слайд

```text
Убери линейный слайд S014. Причина: его смысл можно собрать в соседних экранах без отдельного шага. Содержание: перенести в S013 и S015. Тест: нет. Проведи изменение по системе проекта и обнови все связанные документы.
```

## 3. Пример: разделить один слайд на два

```text
Раздели слайд S015 на два. Первый: Проверка массы и схемы строповки. Второй: Проверка устойчивости и оснастки. Причина: один экран перегружен и в нем смешаны два разных шага проверки. Тест: нет. Проведи изменение по системе проекта и обнови все связанные документы.
```

## 4. Пример: добавить подвальный слайд

```text
Добавь вынос от S015. Тип: P. Содержание: контрольные точки проверки схемы строповки перед подъемом. Причина: основной экран перегружается, если выносить все пояснения в линейный маршрут. Если это PP: не применимо. Обнови связи, возврат и связанные документы проекта.
```

## Как использовать

- если пример подходит почти полностью, копируй как есть и меняй только `1-2` слова;
- если пример не подходит, бери его как основу и заменяй `S-ID`, название и причину;
- если меняется основной маршрут, используй примеры `1`, `2` или `3`;
- если нужен только локальный вынос, используй пример `4`.
